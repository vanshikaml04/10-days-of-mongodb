# MongoDB 10-Day Practice 🚀

A structured **10-day MongoDB practice journey** focused on building strong fundamentals through **daily hands-on queries, real-world exercises, and concept-based practice**.

This repository documents my consistent effort to learn MongoDB from scratch by solving practical database tasks day by day — covering everything from **CRUD operations** to **query operators**, **document structure**, **field naming edge cases**, and more.

---

## 📌 Why I Made This Repository

I wanted to learn MongoDB in a **disciplined, practical, and day-wise manner** instead of just watching tutorials.

So I created this repo to:

- Practice MongoDB **daily**
- Build a strong understanding of **database concepts**
- Write and test queries in **mongosh**
- Learn by doing **real exercises**
- Track my progress publicly on GitHub
- Create a solid reference for future revision

This repository reflects my effort to improve through **consistency, repetition, and hands-on problem solving**.

---

## 🎯 Goals of This Repository

- Master MongoDB fundamentals
- Practice CRUD operations deeply
- Understand document-based data modeling
- Learn how MongoDB handles:
  - field names
  - `_id`
  - duplicate keys
  - case sensitivity
  - projections
  - operators
- Build confidence in writing MongoDB queries without relying on copy-paste
- Create a personal MongoDB revision + interview prep resource

---

## 🛠️ Topics Covered

This repository includes practice on topics such as:

- Database and collection creation
- `insertOne()` and `insertMany()`
- `find()` queries
- Projections
- `updateOne()` / `updateMany()`
- `deleteOne()` / `deleteMany()`
- `_id` behavior in MongoDB
- Duplicate key errors
- `ordered: false` in `insertMany()`
- Case-sensitive string matching
- Field naming edge cases:
  - field names with spaces
  - uppercase field names
  - camelCase field names
- Query operators
- Real-world MongoDB practice exercises
